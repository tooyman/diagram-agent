# Workspace rules

## Git

- **Never push to a remote unless I explicitly tell you to.** No `git push` (and no
  `git push --force`, no pushing tags or new branches) on your own initiative — even if a
  commit, merge, or PR step would normally be followed by a push. Wait for an explicit
  instruction like "push" / "push to origin".
- Committing locally is allowed only when I ask for it. If unsure whether I want a commit,
  ask first.
- Never run history-rewriting or destructive remote operations (`push --force`,
  `reset --hard` on shared branches, deleting remote branches) unless I explicitly request
  them.
